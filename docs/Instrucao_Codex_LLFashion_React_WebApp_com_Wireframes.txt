Pagina 1
 LLFashion Moda
 Instrucao para Codex - React Web
 App Mobile com Wireframes
 Objetivo: criar uma experiencia visual de compra no atacado, inspirada no
wireframe, acessada por link enviado no WhatsApp, com carrinho proprio,
 minimo de R$ 200, frete e Pix via Nuvemshop.
Decisao principal
O WhatsApp Flow nao sera usado como fluxo principal para reproduzir o wireframe. O caminho
correto para chegar ao visual proposto e criar uma aplicacao web/mobile em React. O WhatsApp
sera usado como canal de entrada, comunicacao e envio do link de compra/pagamento.
Use este documento como briefing para o Codex. Antes de implementar, o Codex deve analisar a
aplicacao atual e reaproveitar o que ja existe.

Pagina 2
1. Correcao de estrategia
A proposta anterior tentou aproximar o WhatsApp Flow do wireframe visual. Porem, o Flow nativo
renderiza listas, radios, inputs e botoes padrao do WhatsApp. Ele nao permite criar livremente cards
com imagem, botoes customizados, grid de produtos, layout de ecommerce e carrinho visual como
no wireframe.
Como a loja e de roupas femininas no atacado, a cliente precisa visualizar as pecas antes de comprar.
Por isso, a estrategia correta para reproduzir o wireframe e criar uma aplicacao React mobile-first.
Resumo da decisao
●
Criar uma aplicacao React responsiva/mobile-first baseada no wireframe.
●
A cliente acessa essa aplicacao por um link enviado no WhatsApp.
●
Nao usar o carrinho nativo do catalogo do WhatsApp.
●
Nao usar WhatsApp Flow como fluxo principal de compra.
●
WhatsApp Flow pode existir apenas como opcional/fallback, se fizer sentido.
●
Backend continua controlando estoque, carrinho, minimo de R$ 200, frete e pagamento via
Nuvemshop.
2. Contexto atual da aplicacao
●
A aplicacao ja existe e foi criada anteriormente.
●
Hoje ha integracao com WhatsApp, Nuvemshop e validacao de estoque no backend.
●
O fluxo atual/planejado usava WhatsApp Flow e/ou catalogo do WhatsApp para parte da compra.
●
A loja vende moda feminina no atacado, nao venda avulsa comum.
●
O pedido minimo e R$ 200,00.
●
A cliente precisa montar um carrinho com varias pecas.
●
O pagamento deve continuar pela Nuvemshop, preferencialmente por checkout/link Pix.
3. Problema que queremos resolver
O problema nao e apenas tecnico. E principalmente de experiencia de compra.
●
No WhatsApp Flow nativo, a listagem de produtos fica muito simples e textual.
●
A cliente nao visualiza bem as pecas antes de escolher.
●
Roupa feminina exige foto, cor, modelo, caimento e detalhes visuais.
●
O cliente pode abandonar a compra se tiver que escolher apenas por texto.
●
O Flow nao reproduz o wireframe com cards, fotos, botoes e carrinho visual.
●
Como a loja e de atacado, o carrinho precisa ser forte, claro e sempre visivel.

14. Regras de negocio obrigatorias

15. Frete e pagamento
A forma de envio depende dos dados da cliente e do endereco. Por isso, o frontend deve coletar
dados e endereco antes de consultar frete.
Ponto de investigacao obrigatorio para o Codex
●
Verificar na API/documentacao da Nuvemshop qual e a forma correta de obter formas de
envio para um carrinho/endereco.
●
Verificar se a Nuvemshop permite criar draft order ja com o frete selecionado.
●
Verificar se o draft order retorna checkout_url ou abandoned_checkout_url.
●
Confirmar se o checkout_url permite pagamento Pix pela Nuvemshop.
●
Se nao houver cotacao de frete via API, usar fallback com checkout_url da Nuvemshop para
a cliente finalizar frete e Pix.
Fluxo de frete/pagamento:
Carrinho >= R$ 200
-> cliente preenche dados
-> cliente preenche endereco
-> backend consulta formas de envio
-> cliente escolhe frete
-> backend valida estoque e minimo novamente
-> backend cria draft order/order na Nuvemshop
-> backend recebe checkout_url
-> frontend mostra link para pagamento Pix
-> opcionalmente backend envia link tambem no WhatsApp.
